# specs/ -- Specifications Directory

## Top-Level Files

- **delivery-protocol.md** (v1.1) -- Defines DELIVERY.yaml and REVIEW.yaml artifact schemas. All agent deliveries must produce machine-verifiable DELIVERY.yaml; QA produces REVIEW.yaml. Includes anti-hallucination mechanisms (Section 8).
- **delivery-schema.py** -- Python validators: `validate_delivery()` and `validate_review()` functions. Uses dataclasses + yaml.safe_load, performs checksum verification.
- **integration-contract.md** (v1.0.0) -- Interface contracts between all pipeline engine modules. Defines the 9-module map (models -> loader -> validator -> state -> slot_registry -> gate_checker -> runner -> nl_matcher -> __init__) with exact function signatures the Engineer must implement.

## Subdirectories

### pipelines/
Pipeline definition specs, split into:
- **templates/** -- 6 YAML pipeline templates (standard-feature, research-task, hotfix, quant-strategy, security-hardening, compliance-audit). All v2.0.0 slot-based format with explicit data_flow edges. See `specs/pipelines/templates/.overview.md`.
- **slot-types/** -- 8 SlotType YAML definitions (designer, researcher, implementer, reviewer, approver, deployer, auditor, compliance-auditor). Each defines input/output schemas + required_capabilities. See `specs/pipelines/slot-types/.overview.md`.

### designs/
Design work packages (architecture decisions for specific features):
- **wp-5.1-pipeline-observer-hook-design.md** -- Observer hook pattern design
- **wp-5.4-bootstrap-defaults-design.md** -- Bootstrap defaults design

### research/
Background research informing architecture decisions:
- **pluggable-pipeline-research.md** -- Industry analysis of pluggable pipeline patterns (Airflow, Temporal, Prefect) that informed the slot-based architecture.
